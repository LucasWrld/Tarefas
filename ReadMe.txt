Explicando um pouco sobre esta aplicação basicamente a api funciona de forma ilustrativa, ou seja 
ela não esta conectada com o front-end e esta usando um DB local, o swagger está habilitado para os testes
dos end-points e códigos dos mesmos, a api funciona perfeitamente de forma como eu faria se estivesse conectada
com o front-end.


Da mesma forma o front-end está funcionando perfeitamente de forma que ele consegue agendar, editar e excluir as 
tarefas assim como a api também é capaz.

O projeto de inicialização da api é o principal já programado para rodar em iis express. 
O front-end está rodando com o comando "ng serve -o" diretamente no terminal da pasta clienteapp. 

Qualquer dúvida estou aberto para explicar o projeto espero que goste mesmo eu não atendendo um dos requisitos 
que era conectar o front-end com a api, porém deixei tudo encaminhado para que isso ocorra.
